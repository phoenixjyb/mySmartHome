# 日立中央空调 Modbus 接入与调试说明（V1）

**适用资料：**《Modbus & MiniModbus 协议转换器专用点表（多联机系统用）4.17》  
**项目版本：** smartHouse V3.2  
**日期：** 2026-06-25

## 1. 结论

这份点表已经足以建立项目的**只读发现、室内机映射、状态监测和基础控制骨架**。但它不是完整安装手册，暂时不能写死串口参数、从机地址、功能码或地址偏移。

首期路线：

```text
先只读验证
→ 建立 Y / 系统号 / 地址号 / 房间映射
→ 单功能写入测试
→ 写后回读验证
→ 最后封装为 Home Assistant climate 实体
```

## 2. 已确认的协议行为

- 每台室内机占用 91 个寄存器；`B(Y)=40000+91×Y`。
- 室内机按系统号和地址号从小到大排序，`Y=0…159`。
- `4998=1` 才表示转换器可控。
- `4997` 返回室内机数量，合法范围 1–160。
- `4999=1` 清除缓存并重新搜索，最长可能 40 分钟；只在拓扑变化时使用。
- 同一室内机连续写入应间隔约 500ms。
- 控制点写 `0xFFFF` 表示保持该功能当前状态。
- `4004=64`（十进制）会关闭全部室内机；这是受保护动作。

## 3. 关键寄存器

### 3.1 转换器级

| 地址 | 权限 | 含义 |
|---:|---|---|
| 4004 | R/W | 批量关机 |
| 4997 | R | 室内机数量 |
| 4998 | R | 是否可控 |
| 4999 | R/W | 清缓存并重新搜索 |

### 3.2 室内机级

令 `B=40000+91×Y`：

| 地址 | 含义 | 编码 |
|---:|---|---|
| `B+16` | 机种码 | 低字节 |
| `B+26` | 系统号 | 低字节 |
| `B+27` | 地址号 | 低字节 |
| `B+28` | 开关状态 | bit0 |
| `B+29` | 模式状态 | 位掩码 |
| `B+30` | 风量状态 | 位掩码 |
| `B+31` | 当前设定温度整数 | 低字节 |
| `B+35` | 报警代码 | 低字节；非 0 报警 |
| `B+37` | 当前设定温度 +0.5 | bit0 |
| `B+38` | 出风温度 TO | 低字节 |
| `B+39` | 回风温度 TI | 低字节 |
| `B+78` | 开关控制 | bit0 |
| `B+79` | 模式控制 | 位掩码 |
| `B+80` | 风量控制 | 位掩码 |
| `B+81` | 目标温度整数 | 低字节 |
| `B+82` | 目标湿度 | 部分机型支持 |
| `B+83` | 导风板控制 | 4 组统一控制 |
| `B+85` | 目标温度 +0.5 | 点表读写列存在矛盾，暂不启用写入 |
| `B+87` | 地暖控制 | 点表读写列存在矛盾，首期不启用 |

## 4. 数值编码

### 4.1 模式

```text
0x0001 自动
0x0002 制冷
0x0004 除湿
0x0008 送风
0x0010 制热
0x0100 舒爽
0x0200 舒心睡眠
0x0800 全年除湿
0x1000 净化
```

模式互斥，不应同时置位多个普通运行模式。

### 4.2 风量

```text
0x0001 自动
0x0002 高风
0x0004 中风
0x0008 低风
0x0010 超高风
0x0020 中低风
0x0040 静音
```

### 4.3 温度

点表把温度整数部分和 0.5℃ 标志分开：

```text
目标温度 = low_byte(B+81) + 0.5 × bit0(B+85)
状态温度 = low_byte(B+31) + 0.5 × bit0(B+37)
```

但 `B+85` 在点表的读写类型列显示为只读，与其“控制点”描述矛盾。因此在日立书面确认前，首期只开放整数温度设置。

## 5. 上电和发现状态机

```text
连接 Modbus
   ↓
读取 4998
   ├── 1：继续
   └── 非1：等待并重试，不发送控制
   ↓
读取 4997，确认 1–160
   ↓
逐台读取 B+16 / B+26 / B+27
   ↓
填写“系统号+地址号→房间”映射
   ↓
读取状态点位
   ↓
进入正常轮询
```

不要在每次 Home Assistant 重启时自动写 4999。只有室内机数量、系统号或接线发生变化时，才人工触发重新搜索。

## 6. 房间映射方法

1. 先读取所有 `Y` 的系统号和地址号。
2. 在原厂线控器上一次只操作一个房间，例如开/关或调高 1℃。
3. 观察哪一组状态寄存器发生变化。
4. 将 `Y、系统号、地址号、机种码、房间名称` 写入映射表。
5. 重启转换器后复核映射；排序规则固定时应保持一致。

建议文件：`config/hitachi/hitachi_units.yaml`，示例：

```yaml
units:
  - y: 0
    system: 0
    address: 1
    room: living_room
  - y: 1
    system: 0
    address: 2
    room: bedroom1_office
```

## 7. Home Assistant 接入策略

### 7.1 为什么先不用最终 climate 实体

日立协议具有三个特点：

- 状态和控制位于不同寄存器；
- 模式和风量是位掩码；
- 0.5℃ 使用独立寄存器位。

Home Assistant 的通用 Modbus climate 可以映射温度、模式、风量和开关，但对于这种“状态寄存器与命令寄存器分开 + 半度拆位”的设备，直接配置容易出现状态不一致。因此项目先提供只读 YAML 和受保护写入脚本，再决定使用模板实体还是独立适配服务。

### 7.2 推荐轮询频率

| 数据 | 建议初值 |
|---|---|
| 4998 可控状态 | 5 秒 |
| 4997 数量 | 60 秒或仅启动读取 |
| 开关/模式/风量/温度 | 10–15 秒 |
| 报警代码 | 10–15 秒 |
| 机种/系统号/地址号 | 300 秒或仅启动读取 |

调试完成后可降低静态信息的轮询频率。

### 7.3 写入保护

- `4998 != 1`：拒绝写入；
- 单次只允许一个控制请求进入队列；
- 同一室内机连续写入间隔至少 500ms；
- 每次写入后回读状态寄存器；
- 不自动触发 4999；
- `4004` 批量关机需要二次确认；
- 暂不写 `B+85/B+87`；
- 所有脚本记录发起人、时间、寄存器和值。

- [ ] 建立所有室内机的房间映射；

- [ ] 写后回读并与原厂线控器核对；
- [ ] 验证原厂线控器和 Modbus 同时控制的优先级；

- [ ] 取得报警代码和机种码完整表。

## 9. 当前不能确认的内容

- 串口参数和从机地址；
- FC03/FC06/FC16 的实际支持情况；
- 工具地址是否需要 `-1`；
- 半度写入和地暖控制是否可写；
- 各机型有效温度范围、模式和风量；
- 报警代码含义；
- 转换器一次最大读写长度。

这些项目必须通过转换器安装手册、日立技术确认或现场只读测试补齐，不能凭点表猜测。
