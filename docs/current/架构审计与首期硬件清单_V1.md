# 架构审计与首期硬件清单（V1）

**项目：** mySmartHome / smartHouse  
**日期：** 2026-06-26  
**阶段：** 设计审计、首期硬件选型、装修队布线交底前

## 1. 结论

当前 V3.2 架构方向是正确的：

- 有线骨干、无线终端；
- Home Assistant 作为本地开源控制平台；
- Zigbee2MQTT / MQTT / ESPHome 作为成熟开源基础设施；
- 日立空调和格恩中央加湿走独立 RS-485 通道；
- 安全和基础控制保留原厂路径；
- Storage room 对应新版平面图衣帽间，作为网络/自动化设备间，同时放置已确认的 Bambu P2S 独立打印位。

本轮审计建议做 4 个小调整：

1. **控制主机优先选 mini PC / Mac mini，不把 Raspberry Pi 作为长期主控首选。**
2. **交换机从“16 口够用”升级为“16 口可用、24 口更稳”。** 如果预算和机柜空间允许，首期直接上 24 口可管理 PoE+ 会减少后期端口焦虑。
3. **Zigbee 协调器坚持 Ethernet/PoE 独立放在中央走廊。** 不放机柜，不用插在主控 USB 上。
4. **RS-485 网关按 4 路隔离型采购。** 日立、格恩、电表/水表和未来设备分开通道，不共线。

## 2. 控制主机选择

### 推荐方案

首期主控建议：

```text
mini PC 或 Mac mini
├── 有线 Ethernet
├── NVMe SSD
├── 16 GB RAM 优先
├── UPS 供电
└── Home Assistant OS / VM + 后续独立 Linux 服务
```

不建议把 Raspberry Pi 5 作为长期主控首选，除非只是临时试验。原因：

- 本项目后期会有 Modbus 轮询、历史数据、自研服务、备份、可能的本地语音/AI；
- 长期运行更看重 SSD、散热、备份恢复和虚拟化/容器余量；
- Pi 5 必须配 NVMe、散热、稳定电源和 UPS 后才接近可靠主控形态；
- Pi 5 可以保留为实验机、备份 HA、边缘采集或临时现场调试机。

### 采购建议

| 优先级 | 类型 | 建议 |
|---|---|---|
| P0 | 主控 | mini PC N100/N150/N305 类或已有 Mac mini；16GB RAM；500GB/1TB NVMe |
| P1 | 备份/实验 | Raspberry Pi 5 8GB/16GB + NVMe HAT + SSD + 官方电源 + 主动散热 |
| 不建议 | 纯 microSD Pi | 不作为长期主控 |
| 不建议 | Home Assistant Green | 适合轻量 HA，不适合作为本项目长期自研平台主机 |

## 3. 网络和布线审计

### 3.1 端口容量

当前施工稿估计首期 13-16 个端口。这个估计成立，但余量偏少。真实落地时容易新增：

- 主卧阳台书桌备用网口；
- 门口高位摄像头 + 低位门铃二选一或同时保留；
- NAS；
- 临时调试口；
- NVR 或小交换机上联；
- 后期 Matter/Thread、Z-Wave 或额外网关。

建议：

- 配线架仍用 24 口；
- 交换机最小 16 口可管理 PoE+；
- 如果预算和机柜空间允许，直接采购 24 口可管理 PoE+；
- 未使用墙口不必全部跳接，按需 patch。

### 3.2 交换机规格

最低要求：

- VLAN；
- 端口隔离 / ACL；
- PoE+；
- PoE 总预算不低于 100W；
- 至少 2.5G 上联；
- 至少 2 个 10G/SFP+ 更好；
- 能导出/备份配置；
- 支持固定端口命名或备注。

候选方向：

| 方向 | 适用情况 | 备注 |
|---|---|---|
| TP-Link Omada | 国内采购、性价比、集中管理 | 推荐优先看 |
| UniFi | 统一体验好、可视化好 | 国内采购和售后需确认 |
| MikroTik / OpenWrt + 独立交换机 | 工程控制强 | 配置门槛高 |

### 3.3 AP

建议首期 2 台吸顶 AP：

- AP1：卧室 1/卧室 2/厨房之间走廊；
- AP2：客厅边吊/空调出风口附近可施工区域，尽量向客餐厅开放面靠近；
- AP-R/AP3：本轮取消，不再预留空管或网线。

新版平面和吊顶约束下，客厅并非全屋满吊顶。AP2 不强求安装在客厅几何正中心，允许偏置安装在靠近空调出风口或墙边立柜的局部吊顶/可暗线区域，但要满足：

- AP 本体装在吊顶外表面或顶面可见面，网线暗走，不外露；
- 出线孔由 AP 底座遮住，不在顶面做 86 面板；
- 尽量避免被金属风管、中央空调出风口、灯带驱动、电机、投影幕盒或大面积柜体遮挡；
- 与空调出风口、金属风管、大灯具和强电驱动保持距离，现场能做到 0.5-1 m 更好；
- 如果客厅 AP2 因边吊位置过偏导致阳台/餐厅/沙发区信号不足，入住后再评估补一台有线 AP 或调整 AP2，不依赖无线 Mesh 主回程。

建议规格：

- Wi-Fi 6/6E/7 均可；新购可优先 Wi-Fi 7；
- 每台至少 2.5G Ethernet uplink；
- PoE+ 供电；
- 支持多 SSID 到 VLAN；
- 支持本地控制器或本地管理，不依赖云。

不建议：

- 无线 Mesh 作为主回程；
- AP 放储物间或金属机柜附近；
- 为 P2S 单独增加 AP，先用 AP2 实测。

## 4. Zigbee / Thread 审计

当前策略正确：Zigbee 做低功耗传感器、开关、门磁、水浸、场景按钮和部分窗帘。

建议采购：

- Ethernet/PoE Zigbee coordinator；
- 放中央走廊开放位置；
- 通过固定 DHCP 进入 Automation VLAN；
- 首期只跑 Zigbee，不把 Zigbee 和 Thread 混在同一个 coordinator 上。

候选方向：

| 类型 | 建议 |
|---|---|
| PoE/Ethernet Zigbee coordinator | SMLIGHT SLZB-06 系列、ZigStar UZG-01、TubesZB 等同类产品 |
| USB Zigbee dongle | 仅作为备选或实验；不适合本项目中央走廊位置 |
| Home Assistant Connect ZBT-2 | 官方 USB dongle，适合作为 HA 附近 USB 方案；不符合本项目 PoE 走廊安装优先级 |

## 5. RS-485 / Modbus 审计

当前最关键的是不要把“有 485 口”当作“可集成”。必须有协议、寄存器、串口参数、读写权限和原厂共存说明。

首期采购建议：

- 4 路相互隔离 RS-485 to Ethernet 网关；
- DIN 导轨或壁挂安装；
- 每路独立波特率、校验、停止位、超时、从机地址；
- 支持 transparent RTU over TCP；
- 支持 Modbus RTU to Modbus TCP 更好；
- 固定 IP；
- 配置可导出；
- 明确 A/B/GND、屏蔽、终端电阻接法。

通道分配：

| Port | 用途 | 首期状态 |
|---|---|---|
| 1 | 日立 Modbus/MiniModbus 转换器 | 点表已取得；等待安装手册和现场参数 |
| 2 | 格恩中央加湿 | 已知 9600 8N1；仍需写功能码和地址基准确认 |
| 3 | 未来电表/水表 | 预留 |
| 4 | 未来大金/其他设备 | 预留 |

候选方向：

| 方向 | 适用情况 |
|---|---|
| Moxa NPort / MGate | 工业可靠性优先，预算较高 |
| ZLAN / USR / Waveshare 工业网关 | 性价比和国内采购优先 |

## 6. 装修队必须完成的内容

装修队负责“隐蔽工程和安装条件”，不负责后期软件联调。

必须交付：

- 所有网线星形回储物间；
- 每根线两端编号；
- 每根 Cat6/Cat6A 测试报告；
- AP1、AP2 吸顶点位和 PoE 网线；
- 中央走廊 Zigbee coordinator 点位和 PoE 网线；
- 日立转换器到储物间独立弱电管和屏蔽线；
- 格恩控制器到储物间独立弱电管和屏蔽线；
- 投影电源、Cat6、影音穿线管；
- 入户高位/低位 Cat6 或空管；
- 衣帽间 P2S 独立台面、独立插座、可选备用网口、可选通风/外排路径；
- 开关盒零线和加深底盒；
- 窗帘盒 220V 和弱电空管；
- 水浸点附近设备位置和可检修空间；
- 封墙/吊顶前照片。

| 类别 | 建议数量 | 协议 | 备注 |
|---|---:|---|---|
