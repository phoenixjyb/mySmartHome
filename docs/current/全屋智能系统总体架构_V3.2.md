# 全屋智能系统总体架构（V3.2）

**项目：** 约 100 ㎡，三卧室、一客厅、一厨房、两卫生间、两个阳台  
**网络：** 1000M 光纤宽带  
**核心原则：** 有线骨干、无线终端；本地优先；原厂控制保留；重要功能断网可用；系统由业主自行管理  
**本版日期：** 2026-06-25  
**版本：** V3.2（日立多联机 Modbus 点表纳入基线；大金资料继续等待）

## 0. 本版确认的空间用途

- **书房：办公室/书房**，放置 RTX 3090 工作站，保留 2 个有线网口。
- **儿童房/次卧：可能放置另一台台式机**，保留 2 个有线网口。
- **主卧：不设置固定网口**，智能传感器使用 Zigbee。
- **衣帽间：网络与自动化设备间 + Bambu P2S 独立打印位**。P2S 使用独立台面和独立插座，不接机柜 UPS/PDU；PLA/PETG 不强制外排，但预留通风或外排条件。
- **客厅阳台：保留 1 个备用网口**；如放洗衣机，增加水浸传感器。
- **衣帽间设备区：网络机柜、Mac mini、网关和 UPS 与 P2S 物理分区**；不再放 3090 工作站。
- **客厅：无线投影仪**，但装修时仍预埋电源、Cat6 和影音空管。

# 2. 全屋传感器架构

## 2.1 传感器分级

### A. 独立安全层

这些设备必须在 Home Assistant、网络或电源服务器故障时仍能独立报警或保护：

- 烟雾报警器；
- 燃气报警器；
- 一氧化碳报警器；
- 配电保护和漏电保护；
- 燃气系统原厂安全保护；
- 电动总水阀的手动操作；
- 空调、加湿器原厂控制器。

### B. 本地自动化层

- Zigbee 温湿度、门磁、水浸、人体/存在传感器；
- 可移动 CO₂ 传感器；
- Modbus 空调及加湿状态；
- 用电和用水计量；
- 智能锁状态；
- 排风和灯光执行器。

### C. 厂家附加层

- 冰箱、洗衣机、扫地机、油烟机、P2S 等厂商 App 或云接口；
- 只用于状态、提醒和非关键场景，不作为安全和基础控制依赖。

## 2.3 温湿度传感器布置原则

- 普通房间安装在内墙、离地约 1.2–1.5m 的活动区域；
- 避开阳光直射、空调送风口、窗边、电脑和电视散热口；
- 书房 的传感器不要装在 RTX 3090 机箱排风方向；
- 卫生间传感器安装在干区，不直接被淋浴水冲刷；
- P2S 位传感器放在衣帽间打印位附近空气流通处，不能紧贴打印机出风口；
- 储物间传感器放在机柜进风侧，不装在交换机或 UPS 正上方；
- 厨房温湿度不纳入全屋空调和加湿控制平均值；
- 卫生间、厨房、主卧阳台 的湿度不参与中央加湿器的主控湿度计算。

## 2.4 推荐数量

首期建议：

- 温湿度：8 个；
- CO₂：可选 1–2 个可移动试点，不作为施工固定点位；
- 人体/存在：3–4 个试点，玄关/过道、客厅、书房、卫生间优先；
- 门磁/窗磁：至少 3 个，卧室可开启窗户按需增加；只做开合状态检测，不配置门窗执行器；
- 水浸：6–8 个；
- 独立烟雾报警：客厅/走廊、衣帽间设备区、P2S 位至少各 1 个，厨房外候选；
- 燃气及一氧化碳报警：按灶具、燃气热水器位置和当地规范配置。

## 3.2 AP 和 Zigbee 协调器位置

- **AP1：** 书房、儿童房/次卧、厨房之间的走廊吊顶；
- **AP2：** 客厅边吊/空调出风口附近可施工区域，允许偏置安装，覆盖客餐厅和阳台；
- **AP-R/AP3：** 本轮取消，不预留空管或备用线；
- **Zigbee 协调器：** 中央走廊开放位置，通过 Ethernet/PoE 接入；
- Zigbee 协调器不放进金属机柜，不与 AP 紧贴安装；
- 2.4GHz Wi-Fi 与 Zigbee 信道在安装调试阶段统一规划，不提前把信道写死。

| NAS | 1 | 未来预留 |

所有线路星形回到储物间配线架，中途不得接头。

## 3.4 交换机选择

推荐：

- 16 个 RJ45 端口；
- 至少 8 个 PoE+ 端口；
- PoE 总功率约 100W 或以上；
- 至少 4 个 2.5GbE 端口；
- 最好有 2 个 10G/SFP+ 端口；
- 支持 VLAN、端口隔离、PoE 管理和链路聚合；
- 使用 24 口被动配线架保留扩展余量。

预计首期实际占用约 10–12 个交换机端口，因此 16 口足够。未使用的墙面端口只留在配线架，不需要全部跳接到交换机。

## 3.5 VLAN 和 SSID

| VLAN | 名称 | 设备 |
|---:|---|---|
| 10 | Trusted | 手机、电脑、3090、儿童房/次卧 台式机 |
| 20 | Automation | Home Assistant、MQTT、Zigbee 协调器、RS-485 网关、ESPHome |
| 30 | IoT | 投影、电视、冰箱、洗衣机、扫地机、P2S 等 |
| 40 | Security | 门口摄像头/门铃、未来 NVR |
| 50 | Guest | 访客设备 |
| 99 | Management | 路由器、交换机、AP 管理 |

建议 SSID：

- `Home` → Trusted；
- `Home-IoT` → IoT；
- `Guest` → Guest。

基本规则：

- IoT 不得主动访问 Trusted；
- Trusted 可访问 Home Assistant 和必要的 IoT；
- Automation 仅允许必要的互联网、DNS、时间和更新；
- Security 仅访问 NVR、DNS、时间和管理端；
- Management 只允许管理员设备访问；
- 远程访问通过 WireGuard，不对公网直接开放 Home Assistant、摄像头或路由器后台。

# 4. RS-485 与暖通架构

## 4.1 日立中央空调：当前主接入基线

已取得《Modbus & MiniModbus 协议转换器专用点表（多联机系统用）4.17》。该文件足以确定日立多联机的**设备发现流程、室内机地址公式、主要状态点位和控制点位**，因此项目当前先以日立方案完成软件与网络骨架；大金方案保留为备选，待 DTA120A611 正式资料到达后再做同表比较。

```text
日立多联机系统
      │ 日立内部总线
      ▼
日立 Modbus / MiniModbus 协议转换器
      │ 预计为 Modbus RTU / RS-485；以转换器安装手册为准
      ▼
隔离型 RS-485 转 Ethernet 网关 Port 1
      │ RTU over TCP 透传
      ▼
Automation VLAN
      │
Home Assistant Modbus / 日立 HVAC 适配逻辑
```

### 4.1.1 已确认的连接流程

1. 转换器首次上电会自动搜索空调机组；机组信息会在成功搜索后约 10 分钟内写入本地存储。
2. 每次上位机建立连接后，先轮询寄存器 `4998`；只有读到 `1` 才进入控制阶段。
3. 读取 `4997` 获取室内机数量；合法范围为 1–160。
4. 按室内机排序编号 `Y=0…N-1` 逐台读取机种、系统号、地址号和状态点位。
5. `4999` 写 `0x0001` 会清除已保存机组信息并重新搜索；仅在室内机数量或接线变化时使用，不作为日常启动动作。
6. 重新搜索最长可能需要 40 分钟；写 `4999` 后转换器应答可能延迟约 5 秒。
7. 同一室内机的相邻两条写寄存器命令建议间隔约 500ms；多参数变更优先合并为一次写操作，但必须先确认转换器支持的写功能码和连续写规则。
8. 无回复时按约 500ms 间隔重试；任何控制前必须确保空调原厂系统已完成调试。

### 4.1.2 转换器级寄存器

| 地址 | 权限 | 含义 | 项目用途 |
|---:|---|---|---|
| 4004 | R/W | 批量关机 | 写十进制 64：全部室内机关机；写 0–63：对应冷媒系统全部关机。仅做受保护的应急/离家脚本，不放普通仪表盘按钮。 |
| 4997 | R | 搜索到的室内机数量 | 自动发现数量，期望 1–160。 |
| 4998 | R | 转换器可控状态 | 值为 1 后才允许轮询和控制。 |
| 4999 | R/W | 重新搜索机组 | 写 1 会清除缓存并重新搜索；仅维护时使用。 |

### 4.1.3 室内机地址模型

每台室内机固定占用 91 个保持寄存器。排序编号 `Y` 按“系统号 → 地址号”从小到大排列，范围 0–159。

```text
B(Y) = 40000 + 91 × Y
```

| 功能 | 地址公式 | 权限 | 数据说明 |
|---|---:|---|---|
| 机种码 | `B+16` | R | 低字节 |
| 系统号 | `B+26` | R | 低字节 |
| 室内机地址号 | `B+27` | R | 低字节 |
| 运行状态 | `B+28` | R | bit0：1 运转，0 停止 |
| 模式状态 | `B+29` | R | 位掩码：自动/制冷/除湿/送风/制热等 |
| 风量状态 | `B+30` | R | 位掩码：自动/高/中/低/超高/中低/静音 |
| 设定温度整数部分 | `B+31` | R | 低字节，单位按摄氏度整数理解，需现场核验 |
| 设定湿度 | `B+32` | R | 仅部分室内机支持 |
| 导风板状态 | `B+33` | R | 4 组导风板状态和段位 |
| 报警代码 | `B+35` | R | 低字节非 0 表示报警 |
| 设定温度 +0.5 标志 | `B+37` | R | bit0 为 1 时加 0.5℃ |
| 出风温度 TO | `B+38` | R | 低字节 |
| 回风温度 TI | `B+39` | R | 低字节；可作为空调回风温度，不替代房间独立传感器 |
| 开关控制 | `B+78` | R/W | bit0：1 运转，0 停止 |
| 模式控制 | `B+79` | R/W | 位掩码，模式互斥 |
| 风量控制 | `B+80` | R/W | 位掩码，风量互斥 |
| 目标温度整数部分 | `B+81` | R/W | 低字节；不同机型范围不同 |
| 目标湿度 | `B+82` | R/W | 个别机型支持；不用于代替格恩中央加湿 |
| 导风板控制 | `B+83` | R/W | 当前点表只支持 4 个导风板统一控制 |
| 目标温度 +0.5 标志 | `B+85` | 点表存在歧义 | 位于控制区域且描述为设定，但表格读写列显示 `Rd`；启用半度写入前必须由日立确认 |
| 地暖控制 | `B+87` | 点表存在歧义 | 文本给出写入流程，但表格读写列显示 `Rd`；本项目首期不启用 |

### 4.1.4 模式和风量值

| 模式 | 写入/状态值 |
|---|---:|
| 自动 | `0x0001` |
| 制冷 | `0x0002` |
| 除湿 | `0x0004` |
| 送风 | `0x0008` |
| 制热 | `0x0010` |
| 舒爽 | `0x0100` |
| 舒心睡眠 | `0x0200` |
| 全年除湿 | `0x0800` |

| 风量 | 写入/状态值 |
|---|---:|
| 自动 | `0x0001` |
| 高风 | `0x0002` |

控制点位写 `0xFFFF` 表示该功能保持当前状态。该规则有利于批量写控制块，但首期仍以**只读验证 + 单功能写入 + 写后回读**为主。

### 4.1.5 Home Assistant 实施方式

首期不直接把每台室内机配置成最终 `climate` 实体，而分三步实施：

1. **只读调试：** 读取 4998、4997 和各室内机状态点位，确认地址基准、从机地址、串口参数及房间映射。
2. **安全控制：** 通过受保护脚本写 `B+78/B+79/B+80/B+81`，每次写后读取状态寄存器确认；相邻写入保持至少 500ms。
3. **统一空调实体：** 地址、写功能码、半度控制和命令寄存器回读全部验证后，再用模板实体或独立适配服务封装为房间 `climate`。

原因是该协议把**状态寄存器与控制寄存器分开**，并把 0.5℃拆到独立位；直接套用通用 Modbus climate 配置容易丢失状态或半度信息。

### 4.1.6 尚缺的安装资料

当前点表不是完整通信安装手册，仍须日立供应商确认：

1. 协议转换器准确型号及与最终多联机型号的兼容性；

3. RS-485 波特率、数据位、校验、停止位；

## 4.2 格恩中央加湿：已取得 BG 系列 Modbus 协议

- 协议：Modbus RTU；
- 物理层：RS-485；
- 波特率：9600；
- 串口格式：8N1（8 数据位、无校验、1 停止位）；

### 4.2.1 已知寄存器映射

| 寄存器 | 权限 | 建议实体 | 已知值/含义 |
|---|---|---|---|
| 0x0001 | R/W | 从机地址 | 初始值 1；表中说明设定范围 2–255 |
| 0x0002 | R/W | 加湿器开关 | `0x01` 开机；`0x10` 关机，注意关机不是 0 |
| 0x0004 | R/W | 手动/定时排水命令 | 0 不排水；1 排水，需确认是否自动复位 |
| 0x0005 | R/W | 目标湿度 | 初始值 60；允许范围和步进待厂家确认 |
| 0x0006 | R | 当前室内湿度 | 整数百分比作为首期假设；量程/缩放待确认 |
| 0x0008 | R/W | 送风风速 | AC 风机 1–3；EC 风机 1–5；默认值待确认 |
| 0x0009 | R/W | 风机模式 | 0 手动；1 自动；2 睡眠；3 急速 |
| 0x000A | R/W | 定时排水间隔 | 1–99；单位为小时或天，由主板设置决定 |

| 0x0014 | R | 运行状态字 | D7 EC 风机；D6 电磁输出；D5 UV；D4 风机；D2 进水阀；D1 水泵；D0 湿轮电机 |
| 0x0015 | R | 故障字 | D4 湿度传感器；D3 高水位；D2 水位探头；D1 加水；D0 排水 |

### 4.2.2 接入前仍须确认

- R/W 寄存器使用 FC06、FC16 或两者均支持；
- 地址在实际网关中采用 0 基还是 1 基；
- 目标湿度和当前湿度有效范围、缩放和异常值；
- 排水命令是否自动复位；
- 原液晶控制器与 Modbus 主站能否长期同时工作；
- A/B/GND、屏蔽接地、终端电阻和最大总线长度。

## 4.3 串口网关分配

推荐采用 **4 路相互隔离的工业 RS-485 转 Ethernet 网关**；如果只采购 2 路，也应保留扩展位置。

| 端口 | 系统 | 当前状态 |
|---|---|---|
| Port 1 | 日立 Modbus/MiniModbus 协议转换器 | 点表已取得；串口参数和从机地址待安装手册 |
| Port 2 | 格恩中央加湿 | 已确认 9600 8N1 |
| Port 3 | 未来 Modbus 电表/水表/计量设备 | 可选预留；不假设小区正式计费表可本地读取 |
| Port 4 | 未来大金替代方案或其他设备 | 预留 |

每一路独立配置波特率、校验、超时和从机地址，不把空调、加湿、电表混接在同一条总线上。

## 4.5 控制原则

- 原厂空调线控器继续负责基本控制和保护；

# 6. 系统交互架构

## 6.1 数据流

```text
Zigbee 传感器
      │
PoE Zigbee 协调器
      │
Zigbee2MQTT
      │
Mosquitto MQTT
      │
Home Assistant
      │
      ├── Zigbee 灯光、窗帘、排风
      ├── Modbus 日立空调
      ├── Modbus 格恩加湿
      ├── LAN/Wi-Fi 投影和音响
      ├── Wi-Fi 家电和扫地机
      └── 自编应用、通知和仪表盘
```

## 6.2 典型场景

### 办公室舒适场景

```text
书房 检测到有人
+ 窗户关闭
+ 温度偏高
→ 通过 Modbus 调整日立设定温度
→ 不直接切断空调电源

CO₂ 持续偏高
→ 提醒开窗或短时排风
```

### 中央加湿场景

```text
白天：客厅 + 书房 湿度作为主要参考
夜间：实际有人卧室湿度作为主要参考
卫生间、厨房、主卧阳台 不参与全屋平均

门窗打开
→ 暂停或降低加湿设定

湿度过高或存在结露风险
→ 降低目标湿度
```

### 卫生间排风

```text
检测到有人
→ 开灯

湿度快速上升
→ 开排风

湿度下降并接近基线
→ 延时关闭排风
```

### 观影场景

```text
按下 Zigbee 场景按键
→ 关闭客厅窗帘
→ 调暗灯光
→ 开启投影
→ 开启功放/播放器
→ 选择对应输入源
```

### P2S 打印场景

```text
检测到 P2S 用电或打印状态
→ 按材料和温度/气味策略开启可选通风/排风

打印结束
→ 如开启排风，延时 15–30 分钟关闭

衣帽间/P2S 位温度过高
→ 提醒并提高通风/排风

烟雾报警
→ 独立声响报警
→ 手机通知
→ 按经电气人员确认的安全逻辑处理电源
```

### 漏水保护

```text
任一关键水浸传感器报警
→ 本地声光/手机通知
→ 关闭电动总水阀
→ 记录报警位置和时间
```

### 离家场景

```text
入户门关闭/上锁
+ 全屋无人在设定时间内
→ 关闭非必要灯光
→ 空调转节能设定
→ 加湿器降低目标
→ 关闭窗帘（按用户偏好）
→ 启动扫地机器人（条件满足时）
```

## 6.3 故障降级

| 故障 | 应保持可用的功能 |
|---|---|
| 互联网中断 | 本地灯光、窗帘、空调、加湿、传感器和场景继续运行 |
| Home Assistant 停机 | 墙面开关、窗帘按键、空调原厂线控、加湿原厂控制继续使用 |
| Zigbee 协调器故障 | 原厂空调/加湿、普通家电和手动开关继续使用 |
| Mac mini 重启 | 网络路由不受影响；路由器为独立设备 |
| 交换机故障 | 安全报警仍应独立；重要设备可手动操作 |
| 厂商云服务故障 | 核心灯光、窗帘、暖通和漏水保护不依赖该云服务 |

## 7.3 自编应用接入原则

- 读取状态优先通过 Home Assistant WebSocket/REST API；
- 事件和松耦合控制可使用 MQTT；
- 不建议多个应用同时直接轮询同一条 Modbus 总线；
- 由 Home Assistant 或单一 Modbus 服务统一管理寄存器读写；
- 重要控制命令增加权限、频率限制、超时和状态反馈；
- 自动化配置、脚本和寄存器映射纳入 Git 版本管理。

## 7.4 备份和运维

- 每日自动备份 Home Assistant；
- 每周把加密备份复制到另一台设备或云存储；
- Zigbee2MQTT 数据目录和协调器配置单独备份；
- 路由器、交换机和 AP 配置导出保存；
- 对网关、AP、协调器和服务器设置 DHCP 固定租约；

- 建立低电量、离线和高温告警；
- 每半年测试一次漏水关阀、烟雾报警、UPS 和故障降级。

# 10. 最终基线结论

```text
有线骨干：
Cat6/Cat6A + PoE + 16 口可管理交换机

无线局域网：
2 台有线 AP，Home / IoT / Guest 分网

低功耗智能设备：
PoE Zigbee 协调器 + Zigbee2MQTT + MQTT

暖通：
日立多联机点表作为当前空调接入基线
格恩使用独立 RS-485
Home Assistant 通过 Modbus 统一编排

核心服务器：
Mac mini 运行 Home Assistant OS 虚拟机

空间分配：
书房 = 3090 办公室
衣帽间 = 网络与自动化设备间 + P2S 独立打印位

控制原则：
原厂控制保留，Home Assistant 负责跨系统联动；
核心设备不依赖互联网或单一厂商云平台。
```
