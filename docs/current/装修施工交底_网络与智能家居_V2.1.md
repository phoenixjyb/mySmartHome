# 家庭网络与智能家居装修施工交底书（修订版 V2.1）

**项目概况：** 约 100 ㎡，三卧室、一客厅、一厨房、两卫生间、两个阳台  
**网络条件：** 计划接入 1000M 光纤宽带  
**设计目标：** 有线骨干、无线终端；本地控制优先；系统由业主自行管理；重要设备不依赖单一品牌云平台  
**修订日期：** 2026-06-25  
**本版主要修改：** 书房 确认为办公室并放置 RTX 3090 工作站；新版平面图中 衣帽间 对应衣帽间，作为网络/自动化设备间并放置 Bambu P2S 独立打印位；日立多联机 Modbus 点表纳入基线；格恩中央加湿按独立 RS-485 接入；卧室 1、卧室 2 保留有线网口，卧室 3 不设固定网口。

> 本文件用于装修、强弱电、暖通及智能家居施工交底。所有门洞、家具、投影位置、空调型号、加湿器型号、进水位置和室外排风路径，必须在封墙、吊顶前由业主、设计师及施工负责人现场复核。强电、燃气、暖通和排风施工必须由具备相应资质的人员完成，并遵守当地现行规范。

## 1. 本版方案结论

1. **书房 作为办公室**，RTX 3090 工作站放在 书房，不放储物间。
2. **衣帽间作为 P2S 打印区**，采用独立台面、独立插座、与机柜分区、独立安全监测，并预留通风或外排条件。
3. **衣帽间 同时作为网络与自动化设备间**，放机柜、光猫、路由器、交换机、Mac mini、UPS 和 RS-485 网关。
4. 初期安装 **2 台有线吸顶 AP**；保留第 3 个 AP 的空管或吊顶检修条件即可。
5. 卧室 1、卧室 2 各保留 **2 个有线网口**；卧室 3 不设固定网口。
6. P2S 位可选保留 **1 个备用有线网口**；客厅阳台 保留 **1 个备用有线网口**。
7. 客厅投影仪可使用无线网络，但仍预留 **220V、Cat6 和影音穿线管**。
8. 阳台摄像头不是默认项；入户区域预留 PoE 摄像头或可视门铃线路。
9. 交换机采用 **16 口可管理 PoE+**，24 口被动配线架保留扩展余量。
10. 当前空调接入基线为 **日立 Modbus/MiniModbus 协议转换器**；格恩中央加湿使用独立 RS-485。两者不得共用同一串口通道。
11. 灯光和窗帘保留实体操作；Home Assistant 只作为统一控制和自动化层。

## 2. 网络拓扑

```text
运营商光纤
   │
光猫 / ONT
   │
独立软路由 / 防火墙
   │ 2.5GbE
16 口可管理 PoE 交换机
   ├── AP1：左侧走廊吸顶 AP
   ├── AP2：客厅吸顶 AP
   ├── 卧室 1：2 个有线网口
   ├── 卧室 2：2 个有线网口
   ├── 衣帽间 P2S 位：0-1 个备用有线网口
   ├── 客厅阳台：1 个备用有线网口
   ├── 客厅投影/影音备用网口
   ├── 书房 RTX 3090 工作站
   ├── Mac mini / Home Assistant 主机
   ├── Zigbee 以太网/PoE 协调器
   ├── 日立空调 Modbus 转换器 / RS-485 网关
   ├── 格恩中央加湿 RS-485 网关
   └── 入户 PoE 摄像头或可视门铃（可选）
```

### 2.1 无线覆盖

- **AP1：** 设置在卧室 1/卧室 2/厨房之间的走廊吊顶，覆盖左侧卧室、厨房及储物间。
- **AP2：** 设置在客厅边吊/空调出风口附近可施工区域。客厅没有全屋满吊顶时允许偏置安装，覆盖客餐厅和阳台。
- AP 必须使用网线回程和 PoE 供电，不采用无线 Mesh 回程作为主方案。
- AP 不得安装在储物间、金属机柜、金属风管旁或大型灯具上方。
- AP 本体顶面明装，网线暗走，出线孔由 AP 底座遮住；客厅 AP2 不贴空调出风口、灯带驱动、投影幕盒或大面积柜体。
- Bambu P2S 使用无线网络时，应在衣帽间 P2S 实际台面位置测试 AP1/AP2 的 2.4/5GHz 覆盖。

### 3.1 影音管线

- 投影吊装位置最终根据幕布、投射距离和家具确认。

- 穿线管用于未来光纤 HDMI、USB 延长线或控制线，不应与 220V 强电共管。
- 即使投影初期完全使用 Wi-Fi，备用网线也不需要立即接入交换机。

## 4. 交换机和机柜调整

### 4.1 交换机不需要默认采用 24 口

本项目取消卧室 3 网口及默认阳台摄像头后，初期有效端口预计约 12–15 个，推荐：

- **16 口可管理交换机；**
- 至少 **8 口 PoE+**；
- PoE 总功率建议不低于约 100W；
- 至少 4 个 2.5GbE 端口；
- 最好带 2 个 10G/SFP+ 端口，供 RTX 3090 工作站、Mac mini 或未来 NAS 使用；
- 支持 VLAN、链路聚合、端口隔离和 PoE 管理。

建议仍采用 **24 口被动配线架** 或带余量的模块化配线架。配线架多几个空位不会增加功耗，且便于以后增加网线；交换机则使用 16 口即可。

### 4.2 预计端口占用

| 用途 | 端口数 |
|---|---:|
| 路由器上联 | 1 |
| AP1、AP2 | 2 |
| 卧室 1 | 2 |
| 卧室 2 | 2 |
| 主卧阳台、客厅阳台 | 2 |
| 投影/影音备用 | 1–2 |
| RTX 3090 工作站 | 1 |
| Mac mini | 1 |
| Zigbee 协调器 | 1 |
| RS-485 网关（日立+格恩共用一台多口网关） | 1 |
| 入户摄像头或门铃 | 0–1 |
| **合计** | **13–16** |

未使用的墙面网口不必全部插到交换机上，按需在配线架上跳接即可。

## 5. 设备空间施工要求

### 5.1 衣帽间：网络与自动化设备间

储物间只放：

- 9U/12U 通风机柜；
- 光猫、路由器、16 口 PoE+ 交换机和配线架；
- Mac mini / Home Assistant；
- 2/4 路隔离型 RS-485 转 Ethernet 网关；
- 网络 UPS；
- 温湿度和独立烟雾报警器。

施工要求：

- 机柜不得完全密闭，设置简单高低位通风或静音排风；
- 不放 RTX 3090 工作站；P2S 需与机柜/UPS/配线架/衣物/纸箱/溶剂分区；
- UPS 只带网络设备、Mac mini 和网关；
- 预留独立带地电源、照明和检修空间；
- Zigbee 协调器不得装入金属机柜，应通过 PoE 网线安装在中央走廊。

### 5.2 书房：RTX 3090 办公室

- 书桌位提供 2 个 Cat6A 网口，其中 1 个给工作站；
- 提供足够带地插座，避免长期串联插排；
- 机箱周围保留进排风空间，不放入封闭柜；
- 温湿度传感器避开电脑排风方向；
- 存在传感器可先按候选点位预留；CO2 不作为本轮固定施工点位，后续可用普通插座/USB-C 桌面设备试点；
- 如未来需要远端机房摆放工作站，可预留从 书房 至储物间的大口径弱电空管，但首期不必穿 HDMI/USB 线。

### 5.3 衣帽间：P2S 独立打印位

- P2S 放在衣帽间内独立稳固台面/搁板，保留开门、取件、AMS、耗材更换、poop chute、清洁和维修空间；
- P2S 与网络机柜、UPS、配线架、衣物、纸箱、溶剂/胶水物理分区；
- 设置独立带地插座，不接机柜 UPS，不共用机柜 PDU；
- PLA/PETG 不强制外排，但预留通风或约 100mm 的短距离局部外排路径，用于 ABS/ASA/PC、气味或热量控制；
- 如做外排，排风口设置止回、防虫和可检修条件，并先征得物业同意；
- 设置独立烟雾报警器、温湿度传感器；VOC / PM 传感器可后装；
- P2S 使用 AP1/AP2 的 Wi-Fi；如布线方便，可保留 1 根 Cat6A 作为未来备用，不增设独立 AP；
- 不把打印废气接入中央空调回风、新风送风、厨房烟道或公共卫生间风道。

### 5.4 电气与安全

- 书房 工作站和衣帽间 P2S 不共用低质量移动插排；
- 打印机回路和办公室高负载回路由电气人员按实际负载核算；
- 网络 UPS 不给工作站或打印机供电；
- 如需远程断电，采用合规导轨接触器和人工复位逻辑；
- 烟雾、漏电、燃气和一氧化碳保护必须独立于 Home Assistant。

## 6. 门锁、门铃和门外摄像头

### 6.1 门铃是否必须

门铃不是必须设备，选择取决于楼宇已有对讲系统和生活习惯：

- 若楼宇已有可靠可视对讲，且主要需求只是远程查看门外情况，可采用“**智能门锁 + 高位 PoE 摄像头**”；
- 若经常有快递、访客，并需要对方主动呼叫和双向语音，则“**智能门锁 + PoE 可视门铃**”更方便；
- 单独摄像头可以持续查看或录像，但不能完全替代访客按铃和双向通话体验；
- 智能门锁负责开门和门锁状态，不应把它当作门外视频设备。

### 6.2 本项目建议

装修阶段同时预留：

- 入户门上方高位 Cat6 一根；
- 入户门旁低位 Cat6 或至少空管一根；
- 设备阶段只安装其中一种也可以，未使用线路不接交换机。

门外设备安装前须确认物业要求及公共走廊隐私范围，镜头应尽量避免正对邻居门口或私人区域。

## 7. C1、C2 阳台摄像头调整

原方案中的 C1、C2 只是安防扩展建议，不是必需设备。本版调整如下：

- 删除主阳台和次阳台的默认摄像头；
- 次阳台仅保留 1 个通用网口；
- 主阳台不默认保留摄像头网线；
- 两个阳台门及可开启窗户使用门磁/窗磁；
- 若住宅位于低楼层、阳台与公共平台或相邻住宅可互通，再考虑增加 1 个摄像头，而不是默认安装 2 个；
- 如只是想确认门窗是否关闭，门磁比摄像头更简单、隐私更好、功耗更低。

## 8. 日立中央空调 Modbus 接入要求

### 8.1 当前确认内容

项目已取得日立多联机 Modbus/MiniModbus 点表，可确认：

- 转换器通过 `4998` 表示是否可控；值为 1 才能发送控制；
- `4997` 返回室内机数量，范围 1–160；
- `4999` 写 1 会清除缓存并重新搜索，仅在拓扑变化时人工使用；
- 每台室内机占用 91 个寄存器，基址 `B=40000+91×Y`；
- 可读取开关、模式、风量、设定温度、回风温度和报警；
- 可控制开关、模式、风量和整数温度；
- 同一室内机连续写寄存器应间隔约 500ms。

### 8.2 仍需日立供应商提供

1. 协议转换器准确型号及兼容表；
2. RS-485 波特率、数据位、校验、停止位；
3. Modbus 从机地址；
4. FC03、FC06、FC16 支持情况；
5. 寄存器是否按点表原地址发送，还是网关端需减 1；
6. 机种码和报警代码表；
7. 原厂线控器与 Modbus 同时操作的优先级；
8. 半度温度寄存器 `B+85` 的实际可写性。

### 8.3 施工预留

- 从日立协议转换器可检修位置到储物间敷设独立弱电管；
- 按日立手册使用指定屏蔽双绞线，A/B/GND 和屏蔽端清晰编号；
- 不与格恩加湿共用同一条 RS-485 总线；
- 与 220V 强电分管敷设；
- 转换器和串口网关位置预留电源、散热和检修空间；
- 日立内部总线由日立安装方接线，装修电工不得自行接入。

### 8.4 控制原则

- 保留原厂墙面线控器；
- Home Assistant 首期只读调试，完成房间映射后再开放写控制；
- `4998 != 1` 时禁止控制；
- 不自动触发 4999；
- 批量关机 4004 只做受保护脚本；
- 不通过智能插座切断空调供电。

## 9. 格恩中央加湿系统接入要求

格恩 BG 系列液晶控制器已确认：

- RS-485 / Modbus RTU；
- 9600、8N1；
- 读取功能码 03H；
- 可读当前湿度、水位、运行和故障；
- 可控制开关、目标湿度、排水、风速和风机模式。

施工要求：

- 格恩控制器至储物间敷设独立弱电管和厂家指定屏蔽线；
- 接入多口隔离串口网关 Port 2，不与日立共线；
- 保留格恩液晶控制器独立运行；
- 采购前让厂家补充写功能码、地址基准、A/B/GND、终端电阻和排水命令复位方式；
- 设备附近预留电源、排水、检修空间和水浸传感器。

Home Assistant 只调整目标湿度、场景和告警；基础恒湿、缺水和故障保护仍由格恩原控制器完成。

## 10. 温湿度传感器是否复用

### 10.1 原设备传感器可以复用，但不建议完全依赖

中央空调和中央加湿器自带传感器可用于：

- 设备自身闭环控制；
- 在接口开放时向 Home Assistant 显示当前数据；
- 故障或状态监测。

但它们的安装位置可能位于室内机回风口、吊顶、线控器或风道，读数不一定代表床边、书桌或沙发处的实际体感。因此建议保留少量独立传感器。

### 10.2 推荐独立传感器数量

| 区域 | 建议传感器 |
|---|---|
| 客厅 | 温度、湿度、CO₂ 各 1 组 |
| 卧室 1 | 温湿度 1 个 |
| 卧室 2 | 温湿度 1 个 |
| 卧室 3 | 温湿度 1 个，使用电池 Zigbee/Thread，不需要网口 |
| 卫生间 1、2 | 湿度传感器各 1 个，用于排风联动 |
| 衣帽间 P2S 位 | 温湿度、独立烟雾；VOC/PM 可后装 |
| 储物间 | 温湿度、独立烟雾、UPS 状态 |

控制策略：

- 空调和加湿器原厂控制器负责基本运行和设备保护；
- Home Assistant 根据各房间传感器、人员存在和时段，调整设定值或运行计划；
- 不采用频繁通断电方式控制空调和加湿器；
- 自动化设置温度、湿度回差和最短运行时间，避免设备反复启停。

## 11. 灯光开关智能化改造

### 11.1 装修阶段必须完成的基础条件

每个计划安装智能开关或继电器的 86 底盒必须具备：

- 火线 L；
- 零线 N；
- 对应灯具负载线；
- 地线及保护接地按电气规范执行；
- 加深底盒，建议有效深度 60 mm 左右或按模块尺寸确定；
- 足够检修空间和导线余量。

强电接线必须由专业电工施工。不同漏电保护回路的零线不得混用。

### 11.2 推荐控制方式

优先顺序：

1. **带实体按键的本地智能墙壁开关；**
2. **普通机械开关 + 认证智能继电器模块；**
3. 大规模调光时使用 DALI、0–10V 或厂家明确支持的调光驱动；
4. 不将全屋主照明全部改成依赖云端的智能灯泡。

智能开关/继电器应满足：

- 网络和 Home Assistant 离线时仍可本地开关；
- 支持本地 Zigbee、Thread、RS-485/Modbus 或可验证的本地 LAN 控制；
- 断电恢复后的默认状态可配置；
- 调光模块必须与灯具驱动匹配；
- 电机和灯光回路不得混用普通继电器逻辑。

### 11.3 多控和场景

- 多控位置施工前确认，不采用现场临时拼接方式；
- 可采用一个主负载开关加若干场景面板；
- 入户设置“回家、离家、全关”；
- 客厅设置“会客、观影、清洁、全关”；
- 卧室门口和床头设置“主灯、阅读、起夜、睡眠、窗帘”；
- 卫生间排风保留实体开关，并可由湿度传感器延时控制；
- 重要照明回路必须在服务器故障时仍可使用。

### 11.4 推荐灯光回路划分

最终以照明设计图为准，建议至少分开：

- 客厅主照明；
- 客厅氛围/灯带；
- 投影观影背景灯；
- 各卧室主灯与床头/阅读灯；
- 厨房主灯与操作台灯；
- 两个卫生间照明与排风；
- 玄关和夜间通道灯。

## 12. 电动窗帘远程控制

### 12.1 计划点位

建议至少为以下位置预留：

- 客厅通往主阳台的大窗/门；
- 卧室 1 外窗；
- 卧室 2 外窗；
- 卧室 3 通往次阳台的门窗。

是否安装双层纱帘和遮光帘，由软装方案确定。双层窗帘通常按两台电机和两个控制通道考虑。

### 12.2 施工预留

每个窗帘盒电机端必须预留：

- 220V 常电及保护接地；
- 可检修接线位置；
- 墙面“开、停、关”实体按键或厂家本地控制面板；
- 一根独立弱电空管；
- 如采用 RS-485 电机，按厂家要求敷设屏蔽双绞控制线；
- 窗帘盒不得完全封死，电机需可拆卸维护。

### 12.3 推荐控制方案

按本地可控程度排序：

1. **RS-485/Modbus 窗帘电机：** 经本地网关接入 Home Assistant；
2. **Zigbee/Thread 窗帘电机：** 直接接入本地协调器；
3. **普通开合电机 + 具备互锁功能的认证双路控制模块：** 必须防止开、关方向同时通电；
4. 不优先采用只能通过厂家云端 App 控制的 Wi-Fi 电机。

选型时优先要求：

- 可读取开合百分比或真实位置；
- 支持本地开、停、关；
- 断网可用；
- 停电后可手拉或手动释放；
- 双层窗帘可分别控制。

### 12.4 Home Assistant 中的表现

每组窗帘在 Home Assistant 中作为独立“cover”设备显示，可实现：

- 手机和本地面板开、停、关；
- 百分比位置控制；
- 日出、日落和光照联动；
- 观影模式自动关闭客厅窗帘；
- 夏季强日照时配合空调提前遮阳；
- 离家时关闭指定窗帘；
- 起床场景分阶段打开窗帘。

所有远程控制必须建立在本地可手动操作的基础上。

## 13. 本地智能家居平台

- 主控制平台：Home Assistant；
- 初期主机：24 小时运行的 Mac mini；
- 设备消息：MQTT/Mosquitto；
- Zigbee：Zigbee2MQTT + 以太网/PoE 协调器；
- 自制设备和传感器：ESPHome；
- 日立空调和格恩加湿分别通过独立 Modbus 通道接入；窗帘优先 Zigbee，本地实体按键保留；
- RTX 3090 工作站仅作为 AI、图像、语音、视频或渲染计算节点；
- Bambu P2S 可放入普通 IoT 网络或单独设备网络，优先使用本地局域网模式；
- 远程访问通过 VPN/WireGuard，不直接将 Home Assistant、摄像头或路由器后台暴露到公网。

## 15. 验收要求

1. 所有 Cat6/Cat6A 网线提供测试结果和编号表；
2. 检查网线是否全部独立回到储物间，无暗接头；
3. AP、门口、阳台、空调网关和加湿器网关位置留有可检修余线；
4. 衣帽间 P2S 运行时测试 Wi-Fi、温升、噪声、气味和可选通风/外排效果；
5. 测试储物间机柜通风及 UPS 断电续航；
6. 测试 书房 工作站回路、衣帽间 P2S 回路、接地和保护装置；
7. 测试灯光、窗帘、排风和门锁在 Home Assistant 关闭时仍可手动操作；
8. 测试断网、断电恢复和 UPS 供电状态；
9. 测试日立和格恩在智能网关或 Home Assistant 故障时仍可通过原厂控制器运行；
10. 移交竣工图、线路编号、设备型号、账号归属、协议文档和备份文件。
